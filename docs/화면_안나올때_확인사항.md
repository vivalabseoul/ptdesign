# 🔍 화면이 안 나올 때 확인사항

## ✅ 수정 완료된 항목

1. ✅ Home.tsx에서 로딩 상태 처리 추가
2. ✅ ResultsPage에서 결과가 없을 때 홈으로 이동
3. ✅ AuthContext에서 사용자 프로필 로딩 오류 처리

## 🚀 개발 서버 실행

### 1단계: 개발 서버 실행

```bash
npm run dev
```

### 2단계: 브라우저에서 확인

1. 브라우저에서 http://localhost:3000 접속
2. 브라우저 개발자 도구 열기 (F12)
3. Console 탭에서 오류 확인
4. Network 탭에서 요청 확인

## 🐛 문제 해결

### 문제 1: 화면이 비어 있음

**원인:**
- 로딩 상태가 끝나지 않음
- 인증 상태 확인 중

**해결 방법:**
1. 브라우저 콘솔에서 오류 확인
2. 네트워크 탭에서 요청 확인
3. Supabase 연결 확인

### 문제 2: 로딩 화면만 보임

**원인:**
- AuthContext에서 사용자 프로필 로딩 실패
- 데이터베이스 연결 문제

**해결 방법:**
1. `npm run check-db`로 데이터베이스 확인
2. Supabase Dashboard에서 테이블 확인
3. 브라우저 콘솔에서 오류 확인

### 문제 3: 오류 메시지가 보임

**원인:**
- Supabase 연결 오류
- RLS 정책 오류
- 인증 오류

**해결 방법:**
1. 브라우저 콘솔에서 오류 메시지 확인
2. Supabase Dashboard에서 설정 확인
3. `npm run check-db`로 데이터베이스 확인

## 🔍 확인 방법

### 1. 개발 서버 확인

터미널에서 다음 명령어 실행:

```bash
npm run dev
```

**예상 결과:**
- 서버가 http://localhost:3000에서 실행됨
- 브라우저가 자동으로 열림

### 2. 데이터베이스 확인

터미널에서 다음 명령어 실행:

```bash
npm run check-db
```

**예상 결과:**
- ✅ `users` 테이블이 존재합니다
- ✅ `analyses` 테이블이 존재합니다
- ✅ `payments` 테이블이 존재합니다

### 3. 브라우저 콘솔 확인

1. 브라우저에서 F12 누르기
2. Console 탭 확인
3. 오류 메시지 확인

### 4. 네트워크 확인

1. 브라우저에서 F12 누르기
2. Network 탭 확인
3. 요청 상태 확인
